Lesson 009

Put Claude on Autopilot with /loop

2026-04-01
workflow automation productivity
Quick Quiz
How do you make Claude Code automatically repeat a task every 5 minutes?
Click to reveal answer
Use /loop 5m <your prompt>. Claude converts the interval to a cron job that fires in the background while your session stays open. Tasks auto-expire after 3 days.

Stop Babysitting, Start Looping

You kicked off a deploy. Now you're refreshing the status page every few minutes. Or you've got a PR waiting on CI, and you keep checking back. These are exactly the kinds of repetitive monitoring tasks that /loop was built for.

As shared by @bcherny (Boris Cherny, creator of Claude Code) on X: "/loop and /schedule are two of the most powerful features in Claude Code." He runs multiple loops daily, from babysitting PRs to sweeping post-merge code review comments.

The Syntax

/loop [interval] <prompt or /command>

The interval accepts s, m, h, or d units. If you omit it, the default is 10 minutes. Natural language works too:

# Explicit interval
/loop 5m check if the staging deploy completed

# Natural language
/loop check open PRs for CI failures every 30 minutes

# Chain with a custom command
/loop 10m /babysit

Real Workflows from the Claude Code Team

  • PR babysitting: /loop 5m /babysit — auto-address code review feedback, rebase, and shepherd PRs to merge.
  • Slack feedback: /loop 30m /slack-feedback — automatically create PRs from Slack feedback.
  • Post-merge cleanup: /loop /post-merge-sweeper — address lingering review comments after merge.
  • Stale PR pruning: /loop 1h /pr-pruner — close out stale PRs.
  • Test monitoring: /loop 5m run the test suite and fix failures — continuous green builds.

What You Need to Know

  • Session-scoped: Loops live in your current Claude Code process. Close the terminal, and they stop.
  • Auto-expiry: All scheduled tasks expire after 3 days automatically.
  • Max concurrency: Up to 50 scheduled tasks per session.
  • Jitter: A small random offset (up to 10% of the interval, capped at 15 min) prevents thundering-herd problems.
  • Local timezone: Cron expressions use your local time, not UTC.

Managing Loops

Use /schedule for one-off delayed tasks instead of recurring ones. To see or cancel active loops, ask Claude directly: "list my active loops" or "cancel the deploy monitor loop."

Key Takeaway

Any task you'd repeat manually on an interval — monitoring deploys, babysitting PRs, running tests — is a perfect candidate for /loop. Let Claude watch so you can focus on actual work.
