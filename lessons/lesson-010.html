Lesson 010

Plan Mode: Think First, Code Second

2026-04-01
plan-modeworkflowproductivity
Quick Quiz
What does Plan Mode do differently from a normal Claude Code session, and how do you activate it?
Click to reveal answer
Plan Mode makes Claude read-only — it analyzes your codebase and proposes a plan without editing any files. Activate it by pressing Shift+Tab to cycle modes or typing /plan. Once the plan looks right, switch back to normal mode to execute it.

Why Plan Before You Build

The most common Claude Code mistake isn't a bad prompt — it's skipping the thinking step. You describe a complex refactor, Claude starts writing immediately, hits a wall three files in, backtracks, and burns through tokens undoing its own work. Plan Mode prevents this entirely.

As shared by @bcherny on X (Claude Code's creator): "Start every complex task in plan mode. Pour your energy into the plan so Claude can one-shot the implementation."

How Plan Mode Works

In Plan Mode, Claude operates in read-only. It can read files, search your codebase, and reason about architecture — but it won't edit a single file. This separation of thinking from execution is the key insight.

Activate it two ways:

  • Shift+Tab — cycles through permission modes (normal → auto-accept → plan)
  • /plan — enters Plan Mode directly from the prompt

You'll see ⏸ plan mode on in your prompt when it's active.

The Plan-Then-Execute Workflow

Boris Cherny's workflow is instructive: he starts most sessions in Plan Mode, iterates on the plan until he's satisfied, then switches to auto-accept mode to let Claude execute. Here's the pattern:

  1. Explore — "Look at the auth module and explain how session handling works"
  2. Plan — "Propose a plan to migrate from JWT to session cookies. List every file you'd change and why"
  3. Refine — Go back and forth: "What about the middleware? How will you handle the rollback path?"
  4. Execute — Press Shift+Tab to switch to normal or auto-accept mode, then say "Implement the plan"

When to Use It (and When to Skip It)

Use Plan Mode for:

  • Multi-file refactors and architectural changes
  • Unfamiliar codebases where you want to understand before changing
  • Tasks where getting the approach wrong would waste significant time

Skip Plan Mode for:

  • Single-file bug fixes with clear scope
  • Small, well-understood changes
  • Tasks you've done before and trust Claude to handle

Planning in read-only mode also uses fewer tokens than trial-and-error coding, so it's often faster and cheaper for complex tasks.

Key Takeaway

Use Plan Mode (Shift+Tab or /plan) for any complex task — iterate on the plan until it's right, then let Claude one-shot the implementation.
