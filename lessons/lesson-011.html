Remote Control: Drive Claude Code From Your Phone

2026-04-01
remote-control workflow mobile
Quick Quiz
How do you control a local Claude Code session from your phone without exposing source code to the cloud?
Click to reveal answer
Run /remote-control in your terminal session. Claude Code creates an encrypted outbound connection to Anthropic's API—no open ports on your machine. Then open the Claude mobile app or claude.ai/code to send messages that relay back to your local process.

The Problem: You're Away From Your Desk

You kicked off a big refactor, walked away to grab coffee, and now Claude needs input. Or you're on the couch and realize you forgot to start that migration script. Opening a laptop feels like overkill.

Remote Control solves this. As Boris Cherny (Claude Code's creator) shared on X: he writes a lot of his code from the iOS app as a convenient way to make changes without opening a laptop.

How Remote Control Works

Remote Control bridges your local terminal session with the Claude mobile app or any browser. Your code stays local—only messages travel through Anthropic's API.

Start it from an active Claude Code session:

/remote-control

This displays a session URL and a QR code you can scan with your phone. Open it in the Claude iOS/Android app or at claude.ai/code, and you're connected.

What happens under the hood

  • Your local Claude Code process opens an outbound HTTPS connection to the Anthropic API—no listening ports, no port forwarding
  • Messages from your phone go to the API first, then relay back to your local process
  • Source code never leaves your machine; only conversation messages travel through the bridge

Teleport: Pull Cloud Sessions to Your Machine

Remote Control has a counterpart: /teleport (or /tp). If you started a session on the web or mobile, teleport pulls it onto your local machine so Claude can access your full filesystem and tools.

# From your terminal, pull a cloud session locally
/teleport

You can also start a web session directly from the command line:

# Start a remote web session for your current repo
claude --remote "Fix the auth bug in src/login.ts"

Make It Always On

If you want every session to be remotely accessible, enable it in your config:

/config

Then set "Enable Remote Control for all sessions" to true. Now every session you start is automatically available from your phone.

Requirements

  • Claude Code v2.1.52+ (Remote Control), v2.1.74+ (launching from phone)
  • Claude Max, Team, or Enterprise plan
  • GitHub connected on mobile (being relaxed soon)

Key Takeaway

/remote-control lets you drive a local Claude Code session from your phone or any browser—your code stays on your machine, only messages travel through the bridge.
