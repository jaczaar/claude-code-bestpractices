Output Styles: Change How Claude Code Talks to You

2026-04-06
output-stylesconfigcustomization
Quick Quiz
What are the four built-in output styles in Claude Code, and how do you switch between them?
Click to reveal answer
Run /config and select "Output style" to choose between Default (efficient task completion), Explanatory (adds design insights as it codes), Learning (asks you to write code with TODO(human) markers), and Concise (shorter, direct responses). You can also create custom styles as Markdown files.

The Problem: One Size Doesn't Fit All

Claude Code's default mode is optimized for getting work done fast. But sometimes you're onboarding to a new codebase and want explanations. Other times you're learning and want to be coached. Output styles let you change Claude's communication mode without changing what it can do.

As shared by @bcherny on X: "Run /config and set an output style to have Claude respond using a different tone or format. We recommend enabling the 'explanatory' output style when getting familiar with a new codebase."

The Four Built-in Styles

Run /config and select Output style to pick one:

  • Default — Focused on efficient software engineering. Minimal commentary, maximum output.
  • Explanatory — Adds "Insight" blocks between actions, explaining design choices, trade-offs, and codebase patterns. Perfect for onboarding or code review.
  • Learning — A collaborative, learn-by-doing mode. Claude explains concepts and inserts TODO(human) markers where it wants you to write the code yourself. Great for building real skills.
  • Concise — Shorter and more direct. Strips away extra explanation for experienced developers who just want results.

When to Use Each Style

  • Joining a new team? Switch to Explanatory. Claude will teach you the codebase as it works.
  • Mentoring a junior dev? Use Learning mode. Claude becomes a pair-programming tutor that makes them write the important parts.
  • Shipping fast on familiar code? Stay on Default or switch to Concise to cut the noise.

Create Your Own Custom Style

The built-in styles are just the start. You can create custom output styles as Markdown files:

# Generate a custom style interactively
/output-style:new

# Or create one manually
mkdir -p .claude/output-styles

Create a Markdown file with frontmatter:

---
name: security-reviewer
description: Flag security issues as it codes
---

When making changes, always highlight potential security
implications. Add a "Security Note" section when modifying
authentication, authorization, input handling, or data storage.

Save it to .claude/output-styles/ for the project (shareable via git) or ~/.claude/output-styles/ for all your projects. The style will appear in /config automatically.

How It Works Under the Hood

Output styles modify Claude Code's system prompt. They change how Claude responds (tone, structure, verbosity) but don't restrict what it can do. Your selection is saved to .claude/settings.local.json, so it persists across sessions without affecting teammates.

Note: Explanatory and Learning styles produce longer responses by design, which increases output token usage.

Key Takeaway

Run /config and switch to Explanatory when learning a codebase, Learning when building skills, or Concise when shipping fast — or create your own custom style as a Markdown file to tailor Claude's communication to any workflow.
