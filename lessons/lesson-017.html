Plugins: Extend Claude Code from the Built-in Marketplace

2026-04-13
pluginsskillsextensibility
Quick Quiz
How do you discover and install plugins in Claude Code, and what are the three installation scopes?
Click to reveal answer
Run /plugin to open the interactive browser with Discover, Installed, and Marketplaces tabs. When installing, choose between User scope (across all projects), Project scope (shared with collaborators via git), or Local scope (just you, just this repo).

What Is a Plugin?

A plugin is a package that bundles one or more skills along with optional hooks and MCP server configurations. While individual skills are standalone SKILL.md files (covered in earlier lessons on custom commands and /simplify), plugins are the distribution format — the way skills get packaged, shared, and installed.

As Boris Cherny (@bcherny, Claude Code's creator) shared on X: Anthropic maintains an official plugin marketplace (claude-plugins-official) that ships with Claude Code and is available out of the box.

Browsing and Installing

The fastest way to get started is the interactive browser:

/plugin

This opens a three-tab interface:

  • Discover — browse available plugins from connected marketplaces
  • Installed — see what you have, check status, and manage plugins
  • Marketplaces — view and add plugin sources

Press Enter on any plugin in the Discover tab to install it. Or use the CLI directly:

# Install from the official marketplace
/plugin install frontend-design@claude-plugins-official

# Install from a GitHub repo
/plugin install anthropics/skills/skills/skill-creator

Installation Scopes

When you install a plugin, you pick one of three scopes:

  • User (default) — installed for you across all projects. Lives in your ~/.claude/ config.
  • Project — installed for all collaborators on this repository. The plugin reference is committed to the repo, so your team gets it automatically.
  • Local — installed for you in this repository only. Not shared with collaborators.

To choose a scope interactively, use /plugin and select a plugin — you'll be prompted. For the CLI shortcut, scope is chosen during the install flow.

Adding Third-Party Marketplaces

The official Anthropic marketplace is loaded by default, but you can connect additional sources:

# Add a community marketplace (any GitHub repo with a marketplace.json)
/plugin marketplace add user-or-org/repo-name

# Refresh catalog to get latest plugins
/plugin marketplace update

After adding a marketplace, its plugins appear alongside official ones in the Discover tab.

After Installation

Once installed, the plugin's skills are immediately available — no restart required. If you install mid-session, run /reload-plugins to activate them. Skills from plugins work exactly like built-in ones: they can auto-trigger based on task context, or you can invoke them manually via their slash command.

Notable Plugins Worth Trying

  • frontend-design — Anthropic's flagship plugin (277K+ installs) that gives Claude a real design system, avoiding generic-looking UIs
  • code-simplifier — the agent the Claude Code team uses internally for code review, open-sourced by Boris Cherny
  • skill-creator — a meta-plugin that helps you build your own skills with proper SKILL.md structure

Key Takeaway

Run /plugin to browse, install, and manage plugin packages — use User scope for personal tools, Project scope to share with your team, and third-party marketplaces when the official catalog isn't enough.
