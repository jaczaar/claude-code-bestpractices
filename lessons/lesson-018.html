/btw: Ask Questions Mid-Task Without Breaking Flow

2026-04-15
workflowproductivitycontext-management
Quick Quiz
How can you ask Claude a question while it's in the middle of a task — without interrupting its work?
Click to reveal answer
Type /btw followed by your question. It runs as a side conversation with full context visibility but no tools, so the main task keeps running uninterrupted and your conversation history stays clean.

The Problem: Waiting or Interrupting

You're watching Claude refactor a module and a question pops into your head: "Wait, what pattern is it using for error handling here?" Normally you have two bad options: interrupt the task (losing its momentum), or wait until it finishes (losing your thought). The /btw command gives you a third option.

How /btw Works

Type /btw followed by your question at any time — even while Claude is actively processing a response:

/btw What design pattern is this refactor using?

The answer appears in an overlay. Your main task keeps running underneath, completely unaware that you asked anything. When the overlay closes, you're right back where you were.

What /btw can see

  • Full conversation context — everything Claude has seen or done in this session is visible to your side question
  • No tools — /btw can't read files, run commands, or search the web. It answers purely from what's already in context

The inverse of a subagent

This is the key mental model. A subagent has full tool access but starts with an empty context. /btw is the mirror image: full context, zero tools. Use subagents to go find something new. Use /btw to ask about what Claude already knows.

When to Reach for /btw

  • "Why did it do that?" — Ask about a choice Claude just made without stopping the task
  • "What's the plan from here?" — Check Claude's approach mid-flight
  • "Remind me what X does" — Quick recall on something Claude already read
  • "Is this going to affect Y?" — Sanity-check implications while the work continues

Why It's Cheap

/btw reuses the parent conversation's prompt cache. Since the context is already cached from the main task, the side question adds minimal cost — you're mostly paying for the short answer, not re-sending the whole conversation.

When NOT to Use /btw

If your question requires reading a file, running a command, or searching the codebase, /btw can't help — it has no tool access. For those questions, wait for the task to finish or spin up a separate session.

Key Takeaway

/btw lets you ask questions mid-task without interrupting Claude — full context visibility, no tools, minimal cost. It's the inverse of a subagent.
