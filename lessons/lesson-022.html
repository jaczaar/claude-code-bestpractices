Routines: Claude Code Agents That Run While You Sleep

2026-04-24
routines automation cloud
Quick Quiz
What are the three trigger types for Claude Code Routines, and where do they execute?
Click to reveal answer
Routines can be triggered on a schedule (cron-style), via API (HTTP POST), or by GitHub events (PRs, pushes, issues, etc.). They all execute on Anthropic's cloud infrastructure, so they keep running even when your laptop is closed.

What Are Routines?

A routine is a saved Claude Code configuration — a prompt, one or more repositories, and a set of connectors — packaged once and run automatically. Think of it as a Claude Code session that fires on its own, without you being at your terminal. As @noahzweben shared on X: "In addition to a schedule, you can now trigger templated agents via GitHub event or API — with our infra & your MCP+repos."

Routines launched in research preview on April 14, 2026, and are available on Pro, Max, Team, and Enterprise plans with Claude Code on the web enabled.

Three Trigger Types

1. Schedule

Run on a recurring cadence — hourly, nightly, weekly — or once at a specific future time. Great for nightly test suites, dependency audits, or changelog generation.

2. API

Fire on demand by sending an HTTP POST to a per-routine endpoint with a bearer token. This lets you integrate routines into any external workflow — CI pipelines, Slack bots, or custom dashboards.

3. GitHub Events

Run automatically in response to repository events: pull requests, pushes, issues, check runs, workflow runs, discussions, releases, and merge queue entries. Perfect for auto-triaging issues or reviewing PRs the moment they open.

Setting Up a Routine

Head to the web dashboard at claude.ai/code/routines and click New Routine. The key fields:

  • Prompt: The instruction Claude runs each time — be specific, just like a CLAUDE.md directive
  • Model: Choose between Opus, Sonnet, or Haiku depending on task complexity
  • Repositories: One or more GitHub repos, cloned fresh on every run
  • Environment: Controls network access, environment variables (API keys, tokens), and a setup script that runs before each session (install dependencies, configure tools)
  • Triggers: Pick schedule, API, GitHub, or any combination
  • Connectors: Your connected MCP connectors (Slack, Linear, Google Drive) are included by default

Practical Examples

  • Nightly docs sync: Schedule a routine to scan your codebase for new exports and update API docs every night
  • PR auto-review: Trigger on pull_request.opened to run a deep code review and post comments before any human looks at it
  • Issue triage: Trigger on issues.opened to label, assign, and add context from your codebase automatically
  • Release notes: Trigger on release.published to generate and post a changelog to Slack

Limits and Tips

The current cap is 15 runs per account per day. Repos are cloned fresh each run, so keep setup scripts lean. Create custom environments at claude.ai/settings to pre-configure network access and secrets before attaching them to routines.

Key Takeaway

Routines turn Claude Code from a tool you drive into an agent that works autonomously — triggered by schedules, API calls, or GitHub events, running on Anthropic's cloud while you're away.
