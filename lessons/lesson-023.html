MCP Servers: Connect Claude Code to Slack, Databases, and Any API

2026-04-27
MCP integrations configuration
Quick Quiz
How do you give your whole team the same MCP server connections in Claude Code?
Click to reveal answer
Add servers with claude mcp add --scope project, which writes to .mcp.json at the project root. Commit that file to git, and every team member gets the same MCP tools. Use ${VAR} syntax for secrets so credentials stay out of version control.

What Are MCP Servers?

MCP (Model Context Protocol) servers are external tools that Claude Code can connect to — databases, Slack, Sentry, file systems, or any API that speaks the MCP protocol. Once connected, Claude can query your production database, search Slack threads, pull error logs, and more — all from the same terminal session.

As Boris Cherny (@bcherny on X) shared: "Claude Code uses all my tools for me. It often searches and posts to Slack, runs BigQuery queries, grabs error logs from Sentry." The secret is a committed .mcp.json file shared across the team.

Adding an MCP Server

Use the CLI to add servers. All flags go before the server name, and -- separates the name from the command:

# Add a Postgres database (project-scoped, shared via git)
claude mcp add --scope project --transport stdio db \
  -- npx -y @bytebase/dbhub --dsn "postgresql://readonly:${DB_PASS}@prod.db:5432/analytics"

# Add Slack via HTTP + OAuth
claude mcp add --scope project --transport http slack \
  https://mcp.slack.com/mcp

# Add a personal utility (user-scoped, not shared)
claude mcp add --scope user --transport stdio notes \
  -- npx -y @modelcontextprotocol/server-filesystem ~/notes

The .mcp.json File

Project-scoped servers live in .mcp.json at your repo root. This is the file you commit:

{
  "mcpServers": {
    "slack": {
      "type": "http",
      "url": "https://mcp.slack.com/mcp",
      "oauth": {
        "scopes": "channels:read chat:write search:read"
      }
    },
    "db": {
      "command": "npx",
      "args": ["-y", "@bytebase/dbhub", "--dsn", "${DATABASE_URL}"],
      "env": {}
    }
  }
}

Environment variables use ${VAR} syntax with optional defaults: ${VAR:-fallback}. Secrets stay in each developer's environment, not in the repo.

Three Scopes

  • Project (--scope project) — stored in .mcp.json, shared via git. Use for team-wide tools.
  • User (--scope user) — stored in ~/.claude.json, available across all projects. Use for personal utilities.
  • Local (default) — stored in ~/.claude.json under the project path. Private and project-specific.

Three Transport Types

  • stdio — local process, communicates via stdin/stdout. Best for CLI tools and local servers.
  • http — remote server using Streamable HTTP. Supports OAuth and custom headers.
  • sse — remote server using Server-Sent Events. Similar to HTTP but uses SSE streaming.

Managing Servers

# List all configured servers
claude mcp list

# Get details for a specific server
claude mcp get slack

# Remove a server
claude mcp remove db

MCP tools follow the naming pattern mcp__<server>__<tool>. You can pre-approve tools in your CLAUDE.md with allowedTools: ["mcp__slack__*"] to avoid permission prompts.

Key Takeaway

Commit a .mcp.json to your repo with ${VAR} placeholders for secrets, and every team member instantly gets Claude Code connected to your Slack, databases, and APIs — zero individual setup required.
