Lesson 024

Chrome Extension: Give Claude Eyes for Frontend Work

2026-04-29
chrome frontend workflow
Quick Quiz
How do you connect Claude Code to Chrome so it can visually verify frontend changes?
Click to reveal answer
Install the "Claude in Chrome" extension (v1.0.36+), then run /chrome in Claude Code to connect. Claude can now navigate pages, take screenshots, read console errors, and click elements — creating a visual feedback loop that dramatically improves frontend output quality.

The Problem: Building Blind

When Claude Code generates frontend code, it's working blind. It writes HTML, CSS, and JavaScript without ever seeing the result. As Boris Cherny (@bcherny), the creator of Claude Code, put it: "If you ask someone to build a website but they aren't allowed to use a browser, will the result look good? Probably not."

The Fix: /chrome

The Chrome extension gives Claude actual eyes on your UI. Install the Claude in Chrome extension from the Chrome Web Store (version 1.0.36 or higher), then connect it from Claude Code:

/chrome

That's it. Claude Code now has browser tools. No MCP server setup, no config files — one command.

What Claude Can Do With a Browser

Once connected, Claude can:

  • Navigate and screenshot — open your dev server URL, take a screenshot, and visually inspect the result
  • Read console errors — catch runtime errors, failed network requests, and DOM issues without you copying logs
  • Click and type — interact with your UI to test flows like form submission, navigation, and state changes
  • Compare to mockups — paste a Figma screenshot and ask Claude to build the UI, then it opens the browser to compare

The Workflow: Write, See, Fix

The real power is the iterative loop. Instead of generating code and hoping it looks right, Claude follows this cycle:

  1. Write or edit frontend code
  2. Open the page in Chrome and take a screenshot
  3. Visually verify the result against your description or a mockup
  4. Fix issues and repeat until it looks right

This is the same feedback loop that makes verification tips (lesson 012) so powerful, but applied specifically to visual output.

Tips for Best Results

  • Start your dev server first — Claude needs a URL to navigate to. Run npm run dev before connecting
  • Tell Claude the URL — include it in your prompt: "Build a signup form and verify it at http://localhost:3000/signup"
  • Reconnect if idle — the Chrome extension's service worker can go idle during long sessions. Run /chrome and select "Reconnect extension" to fix it
  • Works in VS Code too — type @browser in the VS Code extension to access the same browser tools

Key Takeaway

Connect Claude Code to Chrome with /chrome so it can see, click, and debug your frontend — turning blind code generation into a visual write-see-fix loop that dramatically improves UI quality.
