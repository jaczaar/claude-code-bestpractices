/effort: Five Levels of Reasoning Depth on Demand

2026-05-01
effort performance advanced
Quick Quiz
What are Claude Code's five effort levels, and which one should you use for most coding tasks?
Click to reveal answer
The five levels are low, medium, high, xhigh, and max. For most coding and agentic work, xhigh is the sweet spot — it gives deeper reasoning than high without the runaway token cost of max.

The Problem: One Size Doesn't Fit All

Not every prompt deserves the same amount of thinking. Asking Claude to rename a variable shouldn't burn the same tokens as designing a database migration strategy. The /effort command lets you dial reasoning depth up or down in real time, matching token spend to task complexity.

The Five Effort Levels

Switch levels mid-session by typing /effort <level>:

  • low — Minimal reasoning. Best for quick answers, formatting, classification, or simple extraction. Fastest and cheapest.
  • medium — Balanced speed and quality. Good for straightforward code generation, tool-heavy workflows, and routine agentic tasks.
  • high — Deep reasoning. The default for Opus 4.6 and Sonnet 4.6. Solid for most development work.
  • xhigh — Extended exploration. The default for Opus 4.7. Best for complex agentic coding: API design, legacy migrations, large codebase reviews, and multi-step reasoning chains.
  • max — Unconstrained reasoning with no token ceiling. Use sparingly for the hardest problems — architectural decisions, subtle bug hunts, or security audits. Session-only by default.

How to Set It

Three ways to control effort:

# 1. Slash command (mid-session, persists across sessions)
/effort xhigh

# 2. CLI flag (per-session)
claude --effort high

# 3. Environment variable (global default)
export CLAUDE_CODE_EFFORT_LEVEL=xhigh

Levels set via /effort or the environment variable persist across sessions — except max, which resets to your default when you start a new session (unless forced via the env var).

When to Shift Gears

Think of effort like car gears:

  • Downshift to low or medium when you're doing bulk operations: renaming variables, formatting files, generating boilerplate, or running simple queries.
  • Stay in xhigh for your normal coding flow — it's tuned for the kind of multi-step, tool-calling work Claude Code does best.
  • Upshift to max when you hit a wall: a tricky race condition, a complex refactor, or when you need Claude to really think through trade-offs.

The key insight: xhigh on trivial work wastes tokens because Claude's adaptive thinking runs longer on ambiguous prompts. Drop to medium or low for simple tasks, and you'll save significant token budget for the moments that actually need deep reasoning.

Key Takeaway

Use /effort to match reasoning depth to task complexity — downshift for simple tasks, stay at xhigh for coding, and upshift to max only when you need Claude to think hardest.
