Agent Teams: Multi-Agent Parallelism with Swarm Mode

2026-03-20
new-capability agent-teams parallelism
How do you enable Agent Teams (Swarm Mode) in Claude Code, and what problem does it solve?
Click to reveal answer
Set CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1 as an environment variable. Agent Teams lets multiple Claude agents work in parallel on independent subtasks, each in its own git worktree with a full 1M-token context window, coordinating through a mailbox system.

What Are Agent Teams?

Released on February 5, 2026 alongside Opus 4.6, Agent Teams (also called "Swarm Mode") is an experimental feature that enables multi-agent orchestration in Claude Code. Instead of one agent doing everything sequentially, you can spin up multiple agents that work in parallel — each with its own isolated git worktree and full 1M-token context window.

How to Enable It

Agent Teams is gated behind an environment variable:

export CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1

Set this in your shell profile or before launching Claude Code. Once enabled, the TeammateTool becomes available, allowing the orchestrator agent to spawn and coordinate teammates.

Orchestration Patterns

Agent Teams supports four orchestration patterns depending on your task:

  • Leader — A single coordinator assigns tasks and collects results. Best for feature implementation where one agent plans and others execute.
  • Swarm — Peer-to-peer collaboration. Agents communicate directly via mailboxes. Good for broad codebase exploration.
  • Pipeline — Sequential handoffs from one agent to the next. Ideal for staged workflows like lint, test, then deploy.
  • Watchdog — One agent monitors and supervises others. Useful for long-running tasks that need oversight.

Key Technical Details

  • Independent worktrees: Each agent gets its own git worktree, preventing file conflicts between parallel agents editing different parts of the codebase.
  • Mailbox system: Agents communicate through message-passing mailboxes, solving the limitation where basic subagents couldn't talk to each other.
  • Task dependency tracking: Agents can declare dependencies, and the orchestrator respects execution order — so agent B waits for agent A to finish if needed.
  • 1M token context per agent: Each teammate operates with a full context window, not a shared or reduced one.

When to Use It

Agent Teams shines on large-scale tasks where parallelism provides real speedup:

  • Implementing a full feature across multiple services in a monorepo
  • Writing comprehensive test suites across different modules simultaneously
  • Multi-service refactors (e.g., updating an API contract across frontend, backend, and docs)
  • Codebase-wide audits where each agent handles a different directory
For simple, single-file tasks, regular subagents via the Task tool are still more efficient. Agent Teams adds overhead that only pays off at scale.

Key Takeaway

Agent Teams lets you parallelize large Claude Code tasks across multiple agents, each with isolated worktrees and full context — enable it with CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1 when you need to divide and conquer.
