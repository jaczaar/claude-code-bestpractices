The Three-Level CLAUDE.md Hierarchy

2026-03-23
CLAUDE.md configuration best-practices
Quick Quiz
What are the three levels of CLAUDE.md, and which one wins when they conflict?
Click to reveal answer
Global (~/.claude/CLAUDE.md), Project (<repo>/CLAUDE.md), and Directory (<subdir>/CLAUDE.md). The most specific level wins: Directory overrides Project, which overrides Global.

One File, Three Scopes

CLAUDE.md is Claude Code's persistent instruction file — loaded automatically at the start of every session. But it's not a single file. It operates at three levels, each with a different scope and purpose.

Global: Your Personal Defaults

Location: ~/.claude/CLAUDE.md

This applies to every project you open with Claude Code. Put truly universal preferences here: communication style, security rules, personal git workflow. Keep it under 30 lines.

## Communication
- Be concise and direct.
- Don't ask for confirmation on routine actions.

## Security
- Never commit secrets or .env files.

If you work in both Python and TypeScript repos, don't put "use TypeScript strict mode" here — it'll confuse Claude in Python projects.

Project: Your Team's Playbook

Location: <repo-root>/CLAUDE.md

This is the primary CLAUDE.md for most teams. Commit it to git so everyone benefits. Describe your tech stack, conventions, build commands, and testing patterns. Target under 100 lines.

## Stack
- Frontend: Next.js 14 (App Router), TypeScript, Tailwind
- Backend: tRPC, Drizzle ORM, PostgreSQL

## Testing
- Vitest for unit tests. Playwright for E2E.
- Run: `pnpm test` (unit), `pnpm test:e2e` (E2E).

Directory: Surgical Overrides

Location: <repo>/<subdir>/CLAUDE.md

Use these in monorepos when a subdirectory has its own conventions. Directory files are only loaded when Claude works within that directory tree, making them efficient and targeted. Keep them under 10 lines.

# packages/db
- All schema changes go through Drizzle migrations.
- Never modify migration files after they've been committed.

How Conflicts Resolve

When the same topic appears at multiple levels, the most specific level wins:

Directory CLAUDE.md  >  Project CLAUDE.md  >  Global CLAUDE.md

If your global says "use semicolons" and your project says "no semicolons," Claude follows the project rule. A directory-level override narrows it further — but only within that directory.

Size Budgets Matter

Every line in CLAUDE.md consumes context tokens on every conversation turn. Research shows instruction adherence degrades as the instruction set grows. Stay within these targets:

  • Global: under 30 lines
  • Project: under 100 lines
  • Directory: under 10 lines each
  • Combined total: under 150 lines

For every line, ask: "Would removing this cause Claude to make mistakes?" If the answer is no, cut it.

Key Takeaway

Use Global for personal defaults, Project for team conventions, and Directory for surgical overrides — the most specific level always wins, so keep each one short and purposeful.
