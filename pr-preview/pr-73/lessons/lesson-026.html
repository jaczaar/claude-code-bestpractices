Custom Status Lines: A Live Dashboard Below Your Prompt

2026-05-04
statuslinecustomizationproductivity
Quick Quiz
How do you add a persistent info bar showing model, cost, and context usage below your Claude Code prompt?
Click to reveal answer
Run /statusline and describe what you want in plain English, e.g. "show model, git branch, cost, and context usage." Claude generates a shell script, saves it, and wires it into your settings automatically.

What Is the Status Line?

The status line is a persistent bar that sits directly below your Claude Code prompt. It updates on every tick and can display anything you want: the active model, session cost, context window usage, git branch, working directory, or custom data from any shell command. As Boris Cherny shared on X, "everyone on the Claude Code team has a different statusline."

The Fast Way: /statusline

The easiest way to set one up is with the built-in command:

/statusline show model, git branch, session cost, and context percentage

Claude generates a shell script, saves it to ~/.claude/statusline.sh, and adds the configuration to your settings. Done.

How It Works Under the Hood

Claude Code pipes a JSON blob to your script via stdin on every tick. Your script reads the JSON, extracts what it needs, and prints a single line of text. That text becomes the status line.

Here's a minimal example using jq:

#!/bin/bash
# ~/.claude/statusline.sh
read -r json
model=$(echo "$json" | jq -r '.model // "unknown"')
context=$(echo "$json" | jq -r '.context_percent // 0')
cost=$(echo "$json" | jq -r '.session.cost // "0.00"')
branch=$(git branch --show-current 2>/dev/null || echo "n/a")
echo "$model | $branch | \$$cost | ctx: ${context}%"

Wiring It Into Settings

If you create the script manually, add this to ~/.claude/settings.json:

{
  "statusline": {
    "type": "command",
    "command": "bash ~/.claude/statusline.sh"
  }
}

Available Fields

The JSON blob includes everything you'd want:

  • model — currently active model name
  • context_percent — how full your context window is
  • session.cost — running dollar cost for this session
  • session.tokens_in / session.tokens_out — token counts
  • cwd — current working directory
  • rate_limits.five_hour — rate limit data (v1.2.80+)

Tips for a Great Status Line

  • Keep it single-line. Multi-line status lines with escape codes are prone to rendering glitches.
  • Use /statusline to iterate. Run the command again with a new description to regenerate the script.
  • Track context aggressively. Watching context_percent climb tells you when it's time to /compact.
  • Combine with worktrees. Add the directory or branch to your status line so you always know which worktree you're in when running parallel sessions.

Key Takeaway

Run /statusline with a plain-English description to get a persistent, auto-updating dashboard below your prompt — no manual config needed.
