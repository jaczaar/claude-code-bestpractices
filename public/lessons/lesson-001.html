Agent Teams: Multi-Agent Orchestration in Claude Code

2026-03-20
new-feature agent-teams parallelism
How do you enable Agent Teams in Claude Code, and what problem does it solve?
Click to reveal answer
Set CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1 as an environment variable. Agent Teams lets Claude spawn multiple agents that work in parallel on independent git worktrees, communicating via a mailbox system — turning large tasks like multi-service refactors into parallelized workflows instead of sequential bottlenecks.

What Are Agent Teams?

Released on February 5, 2026 alongside Opus 4.6, Agent Teams (also called "Swarm Mode") is an experimental feature that brings multi-agent orchestration to Claude Code. Instead of a single agent working through tasks sequentially, you can now have multiple agents collaborate in parallel — each with its own 1M-token context window and isolated git worktree.

How to Enable It

Agent Teams is gated behind an environment variable:

export CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1

Set this before launching Claude Code. Once enabled, Claude gains access to the TeammateTool, which allows it to spawn and coordinate teammate agents.

Four Orchestration Patterns

  • Leader — A single coordinator delegates tasks to worker agents and synthesizes their results. Best for feature implementation where one agent owns the plan.
  • Swarm — Peer-to-peer agents work independently on separate tasks. Best for embarrassingly parallel work like "write tests for 5 modules."
  • Pipeline — Sequential handoffs between agents, where each stage builds on the previous one. Best for workflows like lint → refactor → test.
  • Watchdog — A monitoring agent supervises others, checking for issues or quality problems. Best for long-running tasks that need oversight.

Key Capabilities

  • Independent worktrees — Each agent gets its own git worktree, so they can edit files simultaneously without conflicts.
  • Mailbox system — Agents communicate through message passing, solving the limitation where basic subagents couldn't share findings.
  • Dependency tracking — Agents can declare task dependencies, and the orchestrator respects execution order.
  • 1M token context per agent — Each teammate operates with a full context window, not a subset of the parent's.

When to Use Agent Teams

Agent Teams shines on large-scale tasks where parallelism provides real speedup:

  • Full feature implementation spanning multiple services
  • Multi-file refactors across a monorepo
  • Comprehensive test suite generation
  • Parallel code review of independent modules

For small, focused tasks, a single agent is still faster — the overhead of spawning worktrees and coordinating agents isn't worth it for a one-file fix.

Heads Up

This feature is experimental. Expect rough edges. Each agent consumes its own token budget, so costs scale with the number of agents. Monitor your usage when first trying it out.

Key Takeaway

Agent Teams turns Claude Code from a single-threaded worker into a parallel team — enable it with CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1 when you have large, parallelizable tasks that would otherwise take forever sequentially.
