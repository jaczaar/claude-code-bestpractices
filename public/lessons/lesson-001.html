The /loop Command: Scheduled Autonomous Tasks in Claude Code

2026-03-20
new-capability automation slash-commands
Quick Quiz
What does the /loop command do in Claude Code, and how does it differ from a manual RALPH loop script?
Click to reveal answer
/loop runs a prompt or slash command on a recurring interval (default 10 minutes) within your Claude Code session. Unlike external RALPH loop scripts that survive terminal restarts and suit multi-hour autonomous development, /loop is built-in, requires no setup, and is ideal for lighter recurring tasks like monitoring deploys, babysitting PRs, or running periodic code quality checks.

What Is /loop?

The /loop command, introduced in Claude Code v2.1.71, lets you schedule a prompt or slash command to run repeatedly at a fixed interval. It's Claude Code's built-in answer to the "RALPH loop" pattern — the community technique of wrapping Claude in a bash while true loop for autonomous work.

The syntax is simple:

/loop [interval] <prompt or /command>

The interval accepts s, m, h, or d units. If you omit it, the default is 10 minutes. You can also use natural language:

/loop 5m check the deploy status
/loop 30m /simplify
/loop review open PRs every 3 hours

Practical Use Cases

  • Deploy monitoring: /loop 2m check if the staging deploy completed — get notified the moment it lands.
  • CI babysitting: /loop 10m check open PRs for CI failures and suggest fixes — catch flaky tests without staring at GitHub.
  • Recurring quality checks: /loop 30m /simplify — run the built-in code review skill every half hour as you work.
  • Test monitoring: /loop 5m run the test suite, fix any failures, and commit — a lightweight self-healing loop.

Key Constraints to Know

  • Session-bound: Closing your terminal cancels all scheduled loops. Unlike external RALPH scripts, /loop does not survive restarts.
  • Auto-expiry: Scheduled tasks automatically expire after 3 days.
  • Max concurrency: Up to 50 concurrent scheduled tasks per session.
  • Jitter: A small random delay (up to 10% of the interval, capped at 15 minutes) is added to prevent thundering-herd issues.

When to Use /loop vs. a RALPH Script

Use /loop for lightweight, session-scoped monitoring — deploy checks, PR reviews, periodic linting. Use an external RALPH loop script when you need multi-hour autonomous development that survives terminal restarts, like working through a PRD checklist overnight.

The rule of thumb: if the task is "watch and report," use /loop. If the task is "build the whole feature," use RALPH.

Key Takeaway

/loop turns Claude Code into a lightweight cron for your dev workflow — use it to monitor deploys, babysit CI, and run recurring checks without leaving your session.
