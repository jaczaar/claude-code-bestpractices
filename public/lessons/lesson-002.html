Hooks: Auto-Format, Audit, and Gate Every Tool Call

2026-03-23
hooks settings automation
Quick Quiz
Where do you configure Claude Code hooks, and what are the two hook types that let you run commands before or after a tool executes?
Click to reveal answer
Hooks are configured in your settings JSON files (.claude/settings.json, .claude/settings.local.json, or ~/.claude/settings.json). The two types are preTool (runs before a tool executes) and postTool (runs after a tool completes). Each hook targets a specific tool by name and runs a shell command.

What Are Hooks?

Hooks are shell commands that Claude Code runs automatically before or after specific tool invocations. Think of them as git hooks, but for Claude's tools — every time Claude calls Edit, Bash, Write, or any other tool, your hooks fire.

They live in your settings JSON under the "hooks" key:

{
  "hooks": {
    "preTool": [
      {
        "tool": "Bash",
        "command": "echo \"$(date): Bash call\" >> /tmp/claude-audit.log"
      }
    ],
    "postTool": [
      {
        "tool": "Edit",
        "command": "npx eslint --fix $CLAUDE_FILE_PATH"
      }
    ]
  }
}

The Two Hook Types

  • preTool — Fires before the tool runs. Use it for validation, logging, or blocking operations on sensitive files. If a preTool hook exits with a non-zero code, the tool call is blocked.
  • postTool — Fires after the tool completes. Use it for auto-formatting, linting, notifications, or any cleanup that should follow an edit.

Three Practical Recipes

1. Auto-format after every edit

Run Prettier automatically whenever Claude modifies a file:

"postTool": [
  {
    "tool": "Edit",
    "command": "npx prettier --write $CLAUDE_FILE_PATH"
  }
]

2. Audit log for compliance

Track every tool call with timestamps:

"preTool": [
  {
    "tool": "Bash",
    "command": "echo \"$(date -u +%FT%TZ) BASH\" >> ~/.claude/audit.log"
  }
]

3. Block edits to protected files

Prevent Claude from touching migration files that have already been committed:

"preTool": [
  {
    "tool": "Edit",
    "command": "echo $CLAUDE_FILE_PATH | grep -q 'migrations/' && echo 'Blocked: do not edit committed migrations' >&2 && exit 1 || true"
  }
]

Where to Put Them

Hooks can be configured at three levels, just like other settings:

  • .claude/settings.json — Project-level, committed to git. The whole team gets them.
  • .claude/settings.local.json — Project-level, gitignored. Your personal project hooks.
  • ~/.claude/settings.json — Global. Applies to every project.

Gotchas

  • Hooks run synchronously — they add latency to every tool call they target. Keep them fast.
  • A failing preTool hook blocks the tool. This is powerful for gates but can stall Claude if misconfigured.
  • Debug with --debug hooks to see exactly what fires and when.
  • For complex logic, point the hook at an external script rather than inlining shell one-liners.

Key Takeaway

Add preTool and postTool hooks in your settings JSON to automatically format code, log tool usage, or block dangerous operations — enforcing your team's standards on every single tool call without relying on Claude to remember the rules.
