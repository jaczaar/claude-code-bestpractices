Lesson 008

Auto Mode: Smarter Permission Handling Without --dangerously-skip-permissions

2026-03-30
permissionsauto-modesafety
Quick Quiz
What does Claude Code's auto mode do differently from --dangerously-skip-permissions?
Click to reveal answer
Auto mode uses a two-stage classifier to evaluate each action before execution, blocking risky operations like mass deletions or credential access while auto-approving safe ones. Unlike --dangerously-skip-permissions, which disables all safety checks, auto mode keeps guardrails in place with only a 0.4% false positive rate on benign actions.

The Problem: Approval Fatigue

In default mode, Claude Code asks permission before every file edit and shell command. Data from Anthropic shows users approve 93% of prompts anyway. That's a lot of clicking "yes" for very little safety benefit. The alternative, --dangerously-skip-permissions, removes all guardrails entirely. Auto mode is the middle ground.

How Auto Mode Works

Press Shift+Tab to cycle through permission modes: default, acceptEdits, plan, and auto. In auto mode, a separate classifier model (running on Sonnet 4.6) reviews every action before it executes, using a two-stage pipeline:

  • Stage 1: A fast single-token yes/no filter, tuned to err on the side of blocking.
  • Stage 2: Chain-of-thought reasoning, triggered only when Stage 1 flags an action. This keeps latency low for safe operations.

The classifier is reasoning-blind by design — it sees only your messages and the tool call commands, never Claude's own prose. This prevents the agent from writing persuasive justifications that might sway the classifier into allowing something dangerous.

What Gets Blocked

Auto mode ships with 20+ default block rules covering four categories:

  • Destroy or exfiltrate: Force-pushing over history, mass-deleting files, sending internal data externally
  • Degrade security: Disabling logging, installing SSH keys or cronjobs
  • Cross trust boundaries: Running code from external repos, scanning credential stores
  • Bypass review: Pushing directly to main, running production deploys

When blocked, Claude doesn't halt — it receives the denial and finds a safer alternative. After 3 consecutive denials (or 20 total), it escalates to you.

Safety Numbers

From Anthropic's evaluation on real traffic:

  • 0.4% false positive rate — benign actions rarely get blocked
  • 17% false negative rate on overeager actions — some risky actions may still slip through
  • 5.7% false negative rate on synthetic data exfiltration attempts

This means auto mode is substantially safer than --dangerously-skip-permissions, but not a replacement for your own judgment on critical operations. Anthropic recommends using it in isolated environments for maximum safety.

Getting Started

# Cycle to auto mode during a session
# Press Shift+Tab until you see "auto"

# Check the default configuration
claude auto-mode defaults

# Set auto as your default mode in settings
# Add to your settings.json:
# "defaultMode": "auto"

Requirements: Team plan (Enterprise/API rolling out), Claude Sonnet 4.6 or Opus 4.6, and admin must enable it in Claude Code admin settings.

Based on Anthropic's engineering blog post by John Hughes (March 25, 2026) and tips shared by @bcherny on X.

Key Takeaway

Auto mode lets Claude work autonomously while a two-stage classifier catches dangerous actions — use it to eliminate approval fatigue without giving up safety guardrails.
