Lesson 013

NO_FLICKER Mode: Kill Terminal Flashing With One Env Var

2026-04-03
rendering terminal performance
Quick Quiz
How do you enable Claude Code's flicker-free fullscreen renderer?
Click to reveal answer
Run CLAUDE_CODE_NO_FLICKER=1 claude. This switches to an alternate-screen renderer (like vim or htop) that eliminates flickering, keeps memory and CPU flat in long sessions, and adds mouse support.

The Flicker Problem

If you've used Claude Code in long sessions, you've probably noticed the terminal flickering, jumping, or getting sluggish as the conversation grows. The default renderer redraws the entire scrollback on every update, which means more messages = more work for your terminal.

As of April 2026, there's an official fix. Boris Cherny (Claude Code's creator) announced NO_FLICKER mode — an experimental fullscreen renderer that rewrites how Claude Code draws to your terminal.

How to Enable It

One environment variable is all it takes:

CLAUDE_CODE_NO_FLICKER=1 claude

To make it permanent, add this to your shell profile (~/.bashrc, ~/.zshrc, etc.):

export CLAUDE_CODE_NO_FLICKER=1

Requires Claude Code v2.1.89 or later.

What Changes

The new renderer draws on the terminal's alternate screen buffer — the same technique vim, htop, and less use. Instead of dumping output into your scrollback, it virtualizes the entire viewport and only renders what's currently visible.

  • No flickering or jumping — the screen updates smoothly instead of redrawing everything
  • Flat memory and CPU — resource usage stays constant no matter how long your conversation gets
  • Mouse support — scroll, click, and interact using your mouse directly in the terminal

Tradeoffs to Know

Since the UI lives on the alternate screen, a few things work differently:

  • No native Cmd+F / Ctrl+F — your terminal's built-in search won't work on the alternate buffer. Use Ctrl+O then / to search the transcript, or Ctrl+R for reverse search.
  • Scrollback isn't in your terminal history — when you exit, the conversation disappears from your terminal (it's still saved by Claude Code). This is the same behavior as quitting vim.

Most Anthropic internal users already prefer it over the default renderer despite these tradeoffs. If the alternate-screen model feels unfamiliar, give it a few sessions — the smoothness and stability are worth the adjustment.

Key Takeaway

Set CLAUDE_CODE_NO_FLICKER=1 in your shell profile to permanently eliminate terminal flickering and get stable performance in long Claude Code sessions.
