Custom Agents: Build Specialized AI Teammates in .claude/agents

2026-04-17
agents customization advanced
Quick Quiz
How do you create a custom agent in Claude Code, and how is it different from a custom slash command?
Click to reveal answer
Drop a Markdown file with YAML frontmatter in .claude/agents/. Unlike slash commands (which are prompt templates), custom agents get their own identity, tool restrictions, permission mode, model override, and even a color — making them purpose-built specialists you launch with claude --agent=<name>.

Why Custom Agents?

Claude Code's creator Boris Cherny called custom agents "a powerful primitive that often gets overlooked" in his recent thread on hidden features. While custom slash commands (lesson 006) give you reusable prompt templates, custom agents go further: they define an entirely separate persona with its own system prompt, tool access, model, and permissions. Think of them as specialized teammates you can spin up on demand.

Creating Your First Agent

Create a Markdown file in your project's .claude/agents/ directory. The YAML frontmatter configures the agent; the Markdown body becomes its system prompt.

# .claude/agents/security-reviewer.md
---
name: Security Reviewer
description: Reviews code for OWASP Top 10 vulnerabilities
model: claude-opus-4-6
tools:
  - Read
  - Grep
  - Glob
  - WebSearch
disallowedTools:
  - Edit
  - Write
  - Bash
permissionMode: plan
color: red
---

You are a security-focused code reviewer. Analyze code for
OWASP Top 10 vulnerabilities. Never modify files — only
report findings with severity, location, and remediation.

This agent can read and search code but cannot edit or execute anything — perfect for a read-only security audit.

Key Frontmatter Fields

  • tools / disallowedTools — Whitelist or blacklist specific tools. A docs-writer agent might only get Read and Write; a reviewer gets only read tools.
  • model — Override the model per agent. Use claude-haiku-4-5 for fast triage agents, claude-opus-4-6 for deep analysis.
  • permissionMode — Set to plan for read-only agents, or auto for trusted automation agents.
  • color — Visual identity in the sidebar so you can tell agents apart at a glance.
  • maxTurns — Cap how many turns the agent takes before stopping.

Running Agents

Launch a custom agent from the command line:

# Start Claude with a specific agent
claude --agent=security-reviewer

# Or browse and launch interactively
/agents

The /agents command opens a tabbed interface where you can browse, create, and edit agents without touching files directly.

Two Scopes: Personal and Team

  • User scope (~/.claude/agents/) — Available across all your projects. Great for personal utility agents like a "explain this to me" learning agent.
  • Project scope (.claude/agents/) — Lives in your repo. Check it into git so your whole team shares the same specialized agents. A monorepo might have agents for frontend, backend, infra, and docs — each with appropriate tool access.

Practical Agent Ideas

  • PR Shepherd — tools: Bash, Read, Edit; auto-addresses review comments, rebases, and re-runs CI
  • Docs Writer — tools: Read, Write, Glob; model: haiku for speed; generates and updates documentation
  • Migration Bot — permissionMode: auto; handles repetitive refactors across files with minimal supervision

Key Takeaway

Custom agents in .claude/agents/ let you build purpose-built specialists with locked-down tools and permissions — check them into git to give your whole team the same AI teammates.
