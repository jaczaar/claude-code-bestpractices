/voice: Code by Talking Instead of Typing

2026-04-22
voiceworkflowproductivity
Quick Quiz
How do you enable voice input in Claude Code, and what are the two listening modes?
Click to reveal answer
Run /voice to toggle voice mode on. Then choose push-to-talk (hold Space, release when done) for noisy environments, or hands-free mode (Claude listens continuously and responds to natural pauses) for quiet spaces. You can mix voice and typed text in the same prompt.

Why Talk to Your Terminal?

Boris Cherny, the creator of Claude Code, shared that he does most of his coding by speaking rather than typing. It sounds wild until you try it: describing what you want in natural language is often faster than typing a detailed prompt, especially for complex refactors or architectural decisions where you're thinking out loud anyway.

As @bcherny shared on X: voice mode turns Claude Code into a conversational coding partner you can talk to like a colleague sitting next to you.

Getting Started

Toggle voice mode with a single command:

/voice

That's it. Run it again to turn voice off. Voice mode is available to Pro, Max, Team, and Enterprise users on both the CLI and Desktop app.

Two Listening Modes

  • Push-to-talk: Hold the Space bar while speaking, release when done. Best for busy offices or noisy environments where you don't want ambient sound triggering input.
  • Hands-free: Claude listens continuously and detects natural pauses to know when you've finished. Best for quiet spaces where you want a truly conversational flow.

On Desktop, use the voice button instead of the Space bar. On iOS via Remote Control, enable dictation in your device settings.

Mix Voice and Text in the Same Prompt

You don't have to go all-in on voice. The real power move is combining both: speak the high-level intent, then type precise details like file paths, variable names, or code snippets. For example:

  1. Speak: "Refactor the user authentication module to use JWT tokens instead of session cookies"
  2. Type: Focus on src/auth/middleware.ts and src/auth/session.ts

This hybrid approach plays to each input method's strengths: voice for intent and context, text for precision.

When Voice Really Shines

  • Architectural discussions: Talk through design tradeoffs like you would with a colleague — "What if we split this service into two microservices? Walk me through the pros and cons."
  • Code reviews: Narrate what you see as you scroll through a diff — faster than typing paragraph-length feedback.
  • Rubber-duck debugging: Explain the bug out loud. Claude listens, asks questions, and often spots the issue while you're still describing it.
  • Accessibility: Hands-free coding opens up new workflows for developers who find extended typing difficult.

Cost: Zero

Transcription is completely free and doesn't count against your rate limits. The only tokens consumed are the transcribed text sent as a normal prompt — the same as if you'd typed it.

Key Takeaway

Run /voice to talk to Claude Code instead of typing — mix voice for intent with text for precision, and transcription costs nothing.
