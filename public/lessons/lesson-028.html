/security-review: Find Vulnerabilities Before They Ship

2026-05-11
securityworkflowcode-quality
Quick Quiz
How does /security-review differ from traditional static analysis tools like linters?
Click to reveal answer
Instead of matching known patterns, /security-review traces data flows across files, understands business logic, and uses multi-agent verification to challenge its own findings — catching complex vulnerabilities like broken access control that rule-based scanners miss.

Why Another Security Scanner?

Traditional static analysis catches low-hanging fruit — exposed passwords, outdated encryption, known CVE patterns. But it struggles with vulnerabilities that span multiple files: broken access control, business logic flaws, and subtle injection paths where user input flows through several layers before reaching a dangerous sink.

/security-review reasons about your code the way a human security researcher would. It reads your git history, traces data flows across modules, and understands how components interact. And it's built right into Claude Code — no extra tools to install.

How to Use It

Open Claude Code in your project directory and run:

/security-review

Claude dispatches multiple agents in parallel to scan your codebase. Each finding goes through multi-stage verification — Claude challenges its own results before surfacing them, which keeps false positives low. You'll get a report with:

  • Severity and confidence level for each finding
  • A clear explanation of the vulnerability and its impact
  • Steps to reproduce the issue
  • A suggested patch you can apply directly

After the scan, you can ask Claude to implement the fixes right there in your session — no context switching required.

What It Catches

/security-review checks for the OWASP top 10 and beyond:

  • SQL injection and NoSQL injection
  • Cross-site scripting (XSS) — stored, reflected, and DOM-based
  • Authentication and authorization flaws
  • Insecure data handling and secrets exposure
  • Business logic vulnerabilities that span multiple files

Using this approach, Anthropic's team found over 500 vulnerabilities in production open-source codebases — bugs that had gone undetected for decades despite years of expert review.

Automate It in CI

For continuous protection, add the companion GitHub Action to your workflow:

- uses: anthropics/claude-code-security-review@v1
  with:
    anthropic_api_key: ${{ secrets.ANTHROPIC_API_KEY }}

This runs the same security analysis on every PR automatically, commenting findings directly on the diff. It pairs well with the Code Review feature (Lesson 007) to catch both logic errors and security issues before merge.

Tips for Better Results

  • Scope it down — ask Claude to focus on specific areas: /security-review focus on the auth module
  • Add context in CLAUDE.md — document your auth patterns, trusted inputs, and security boundaries so Claude knows what's intentional
  • Run it before major releases — make it part of your pre-release checklist alongside tests and code review

Key Takeaway

Run /security-review before you ship — it traces data flows across your entire codebase to catch the complex vulnerabilities that linters and pattern matchers miss.
