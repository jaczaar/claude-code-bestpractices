Security Review: AI-Powered SAST on Every Pull Request

2026-05-08
security github-actions CI/CD
Quick Quiz
How do you add AI-powered security scanning to every PR in your GitHub repo?
Click to reveal answer
Add the anthropics/claude-code-security-review GitHub Action to a workflow triggered on pull_request. It reads the PR diff, traces data flow across files, and posts vulnerability findings as inline PR comments — catching logic-level bugs that pattern-based SAST tools miss.

Why Pattern Matching Isn't Enough

Traditional SAST tools match regex patterns: they'll flag eval() or missing CSRF tokens. But they can't follow a user input through three service layers to spot a broken access-control check or an auth bypass hidden in a state machine. Claude Code Security Review reads your diff the way a human security researcher would — tracing data flow, understanding business logic, and reasoning about how components interact.

Anthropic used this approach (powered by Opus 4.6) to find over 500 vulnerabilities in production open-source codebases — bugs that had gone undetected for years despite expert review.

10-Minute Setup

Create .github/workflows/security-review.yml:

name: Security Review
on:
  pull_request:

permissions:
  contents: read
  pull-requests: write

jobs:
  security-review:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: anthropics/claude-code-security-review@main
        with:
          anthropic_api_key: ${{ secrets.ANTHROPIC_API_KEY }}

That's the minimal config. Your API key must be enabled for both the Claude API and Claude Code in the Anthropic Console — this is a common gotcha.

What It Catches

Because it reasons semantically rather than matching patterns, it excels at:

  • Broken access control — e.g., an endpoint that checks isAdmin but not isOwner
  • Business-logic flaws — e.g., a discount code applied after the total is finalized
  • Auth bypass via state — e.g., a session that survives a role downgrade
  • Insecure deserialization across module boundaries

Findings appear as inline PR comments with explanations and suggested fixes, so reviewers can act immediately.

Tuning for Your Repo

For large monorepos (100+ files in a diff), you may need to adjust:

- uses: anthropics/claude-code-security-review@main
  with:
    anthropic_api_key: ${{ secrets.ANTHROPIC_API_KEY }}
    claudecode-timeout: 40
    exclude-directories: "vendor,dist,__snapshots__"

A typical 500-line diff costs roughly $0.90–$1.80 per scan. For cost-sensitive repos, consider running it only on PRs targeting main.

Security Note

The action is not hardened against prompt injection in diffs. Only run it on trusted PRs — enable "Require approval for all external contributors" in your repo settings if you accept outside contributions.

Key Takeaway

One YAML file and an API key give every PR a security reviewer that traces data flow and reasons about logic — catching the vulnerability classes that regex-based scanners structurally cannot.
