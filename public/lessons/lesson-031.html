Channels: Message Claude Code From Telegram, Discord, or iMessage

2026-05-15
channelsintegrationsadvanced
Quick Quiz
How do you push messages from Telegram or Discord into a running Claude Code session?
Click to reveal answer
Install a channel plugin with /plugin install telegram@claude-plugins-official, configure your bot token, then restart Claude Code with claude --channels plugin:telegram@claude-plugins-official. Messages you send to the bot arrive in your session, and Claude replies back in the same chat.

What Are Channels?

Channels are MCP servers that push events into your running Claude Code session. Unlike Remote Control (which lets you drive a session from the web), channels create a two-way bridge between a messaging app and your terminal. You text Claude from Telegram while walking the dog, and it reads files, runs commands, and replies right in the same chat—all against your real local codebase.

Three platforms ship as official plugins: Telegram, Discord, and iMessage. You can also build custom channels for CI webhooks, error trackers, or deploy pipelines.

Quick Setup: Telegram in 5 Minutes

Create a bot via BotFather on Telegram and copy the token. Then:

# Install the plugin
/plugin install telegram@claude-plugins-official
/reload-plugins

# Save your bot token
/telegram:configure <your-token>

# Restart with the channel enabled
claude --channels plugin:telegram@claude-plugins-official

Send any message to your bot in Telegram. It replies with a pairing code. Back in Claude Code, run:

# Pair your account and lock it down
/telegram:access pair <code>
/telegram:access policy allowlist

That's it. Only your Telegram account can now push messages into this session.

Discord and iMessage

Discord follows the same pattern: create a bot in the Developer Portal, install with /plugin install discord@claude-plugins-official, configure with /discord:configure <token>, and restart with --channels.

iMessage is even simpler—no bot token needed. On macOS, grant Full Disk Access to your terminal, install the plugin, and text yourself. Self-chat bypasses access control automatically.

Security: Sender Allowlists

Every channel maintains an allowlist. Only paired sender IDs can push messages; everyone else is silently dropped. Being listed in .mcp.json isn't enough to receive events—a server must also be named in --channels. For team deployments, admins control availability with the channelsEnabled managed setting.

Beyond Chat: Webhook Receivers

Channels aren't just for texting. You can build a custom channel that listens on a local HTTP port and forwards webhook payloads from CI, Sentry, or deploy pipelines straight into Claude's session. Claude already has your files open and remembers what you were debugging—the alert arrives in context.

Key Takeaway

Channels turn Telegram, Discord, and iMessage into two-way bridges to your local Claude Code session—text a question from your phone and get answers powered by your real codebase.
